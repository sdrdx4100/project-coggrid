# RPG基盤 0.2

## 画面遷移

1. タイトルからNEW GAMEまたはCONTINUEを選ぶ
2. COGTOWN中央研究所フィールドを探索する
3. メニューでロスター2機のパーツを編成する
4. 研究員へ隣接して会話し、ロボトルを開始する
5. ロボトル終了または撤退後、同じフィールド座標へ戻る

## フィールド

- 18×13の図形タイルマップ
- 矢印キー／WASDによる4方向移動
- 外周壁、障害物、NPCの衝突判定
- Enter／Spaceによる隣接NPCとの会話
- 素材を使用せず、床・壁・人物をCanvasItem描画で表現

## メニューと永続データ

`GameData`が次のRPG状態を所有します。

- フィールド上の主人公座標
- ロボトル勝利数
- チームロスター
- 各機体の4部位ロードアウト
- 各メダルの経験値（レベルは経験値から算出）
- 所持パーツと所持数

セーブデータは`user://save.json`へバージョン付きJSONとして保存します。バージョン1のセーブは経験値0として読み込み、バージョン2へ移行できます。ロードアウトはパーツIDで保存され、`PartCatalog`の定義を参照します。

## メダルの成長

- レベルは1〜99。戦闘終了時に勝利30、敗北12の経験値を参加メンバー全員が得る
- 成功値だけが5レベルごとに+0.1され、Lv99でも最大+1.9に留まる
- 威力、装甲、AP供給、推進、回避、防御にはレベル補正を加えない
- 味方オートAIはLv20で「一般」から「強敵」相当へ成長し、「ボス」は敵専用とする
- メダフォース候補はLv1、Lv20、Lv40で順に解放する。発動処理と選択UIは後続実装とする

## 戦闘との境界

戦闘開始時に`GameData.player_loadouts()`を`BattleState.setup_demo()`へ渡します。戦闘側は受け取ったパーツIDから装甲、行動、推進、回避、防御、外見を構築します。これによりフィールドメニューで行った編成がそのままロボトルへ反映されます。
